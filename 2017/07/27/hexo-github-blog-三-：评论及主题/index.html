hexo+github blog(三)：评论功能

Posted on 2017-07-27

[瞎bb] 博客不仅是个人成长的历程见证，同样也是大家互相分享交流以共同进步的平台，所以为自己的博客加上评论功能也是必不可少的。而在我搭建博客伊始，正赶上国内评论系统龙头 多说 宣布停止服务，老用户在默默唏嘘之余也在积极寻求新的替换品，而我也偷偷跟在他们背后物色合适的评论系统(嘘)。从中我挑了几个比较主流的在这里作简单的介绍，大家按需选择就好~鉴于我暂时只用过hexo，所以本文主要以hexo静态博客说明。详细的参考资料在文末，大家也按需自取吧。

  1. 国内产品

    国内评论系统最大的好处就是可以支持国内社交账号登录，无需翻墙或注册新账号，交流互动更加省心，但是因为有了多说这个前车之鉴，我也不知道国内的产品可以走多远，为了求稳，我还是没有选择以下评论系统。

    • 搜狐畅言

      • 支持其他评论平台的数据导入，支持第三账号登录，支持自定义样式等
      • 需要注册域名并备案，不然只有15天的使用期
      • 使用方法：
        • hexo部分主题(如next)已经集成畅言插件，在themes/next/_config.yml畅言对应的设置处写上自己的APPID和APPKEY即可，
        • hexo其他主题的可以参考这个http://www.zhaiqianfeng.com/2017/03/changyan-for-hexo-next-theme.html,原理是一样的~
        • 对于自己写的页面也可以直接写入官网提供的通用代码，其他评论服务的接入基本都是类似下面就不介绍啦，官网也会有详细的介绍。

    • 友言
      无需备案，可将评论一键同步到各大社区网站(emmm..它的官网总让觉得很虚，有种快垮(?)的感觉..)

  2. disqus

    业界大佬，听说已经跃升全球第二的占有率
    • 界面简洁大气
    • 支持ssl，数据转移方便
    • 只可惜评论需要翻墙和不能自定义样式，不过现在大都具备科学上网的能力，所以问题也不算大。
      要想更科学使用disqus?看这里：）科学使用disqus

  3. gitment

    适合托管在GitHub Pages上的静态博客
    • 使用github账号登录，基于github提醒功能
    • 无需后端代码，在前端直接引入
    • 支持点赞、@人、代码高亮等，而且页面效果还是很让人心动
      demo： https://imsun.github.io/gitment/
      中文文档： Gitment：使用 GitHub Issues 搭建评论系统
      hexo博客戳这里：Hexo博客框架下Gitment取代多说评论

  4. Staticman

    适合托管在 GitHub Pages 上的 Jekyll 博客

  5. 来必力

    来自韩国,引入方法同国内评论系统。
    • UI比较小清新
    • 最重要的是它支持国内主流社交账号登录，无需注册
    • 评论支持分享图片和视频，可以引用他人评论
    • 邮箱消息提醒功能

  6. ISSO

    需要配置python环境

  7. HashOver

    在php环境下运行，支持 XML、MySQL、SQLite 几种数据储存方式
    HEXO评论从“多说”搬迁到Hashover
    Hashover-PHP开源评论系统
